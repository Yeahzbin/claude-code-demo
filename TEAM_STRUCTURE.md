# NotePro 团队分工文档

## 项目背景

**项目名称**：NotePro - 高级笔记应用
**项目类型**：桌面/Web 跨平台应用
**当前状态**：MVP 已完成，数据存储在本地（localStorage）
**目标**：升级为云端数据库存储，支持多端同步

---


## 角色职责

### 1. 项目经理 (Product Manager)

**核心职责**：
- 优化和完善 PRD 需求文档
- 制定迭代计划和里程碑
- 跟进团队成员工作进度
- 协调资源和解决问题
- 需求变更管理

**具体任务**：
- [ ] 更新 PRD 文档，增加数据库设计部分
- [ ] 梳理所有功能模块优先级
- [ ] 制定 V1.0 - V2.0 迭代计划
- [ ] 建立周会机制和进度汇报模板
- [ ] 创建任务看板（kanban）

---

### 2. UI 设计师

**核心职责**：
- 产品视觉设计
- 交互体验优化
- 设计系统维护

**具体任务**：
- [ ] 设计应用 Logo 和品牌规范
- [ ] 完善各页面线框图
- [ ] 深色/浅色主题色值定义
- [ ] 组件设计规范（按钮、卡片、弹窗等）
- [ ] 动效和过渡动画设计
- [ ] 输出设计稿（Figma/Sketch）

---

### 3. 前端开发

**核心职责**：
- 界面开发与迭代
- 用户体验优化
- 与后端对接

**具体任务**：
- [ ] 技术选型：React / Vue + TypeScript
- [ ] 状态管理方案（Redux/Zustand/Vuex）
- [ ] API 对接封装
- [ ] 响应式布局适配
- [ ] 性能优化（加载速度、渲染性能）
- [ ] 单元测试编写
- [ ] CI/CD 流程搭建

---

### 4. 后端开发

**核心职责**：
- 数据库设计与实现
- API 接口开发
- 数据安全保障

**具体任务**：
- [ ] MongoDB 数据库设计
  - 用户集合
  - 笔记集合
  - 文件夹集合
  - 标签集合
- [ ] Node.js/Express 或 NestJS 框架搭建
- [ ] RESTful API 接口开发
- [ ] 用户认证（JWT）
- [ ] 数据同步逻辑
- [ ] 备份与恢复功能
- [ ] 接口文档编写

---

## 技术栈建议

| 角色 | 技术栈 |
|-----|--------|
| **前端** | React 18 + TypeScript + Vite |
| **UI** | Ant Design / Tailwind CSS |
| **状态管理** | Zustand / Redux Toolkit |
| **后端** | Node.js + Express/NestJS |
| **数据库** | MongoDB |
| **部署** | Docker + CI/CD |

## 迭代计划

### Phase 1: 基础架构（2周）
- [ ] 后端：搭建服务器，连接 MongoDB
- [ ] 后端：用户注册/登录接口
- [ ] 前端：API 请求层封装
- [ ] 前端：登录/注册页面

### Phase 2: 核心功能（3周）
- [ ] 后端：笔记 CRUD 接口
- [ ] 后端：文件夹接口
- [ ] 前端：笔记管理页面
- [ ] 前端：数据从 localStorage 迁移到后端

### Phase 3: 增强功能（2周）
- [ ] 后端：标签功能
- [ ] 后端：搜索功能
- [ ] 前端：标签筛选 UI
- [ ] 前端：全局搜索

### Phase 4: 发布准备（1周）
- [ ] 测试与 bug 修复
- [ ] 性能优化
- [ ] 文档完善
- [ ] 部署上线

## 协作规范

### 代码管理
- 使用 Git flow 工作流
- `main` 分支保护，需要 Code Review 才能合并
- 提交信息规范：`feat:`, `fix:`, `docs:`, `refactor:`

### 沟通机制
- 每日站会（15分钟）
- 周报（周五）
- 即时沟通（Slack/飞书/钉钉）

### 文档维护
- 需求文档：PRD（项目经理维护）
- 接口文档：Swagger/OpenAPI（后端维护）
- 设计稿：Figma（UI 维护）
